สร้างแทร็กซ้ำ

เครื่องมือนี้ทำงานด้วยชั้นข้อมูลที่ค่าเวลาเปิดใช้งาน โดยใช้ จุด หรือ พื้นที่ เป็นตัวแทนของ สิ่งที่เกิดขึ้น ภายในเวลาที่กำหนด สามารถกำหนดได้ในครั้งแรก ด้วยฟีเจอร์ที่ต้องการแทร็คด้วยการใช้การระบุข้อมูล ด้วยการใช้เวลาในแต่ละตำแหน่ง แทร็กจะมีการแสดงตามลำดับ และกลายเป็นเส้น หรือ พื้นที่ แสดงเป็นเส้นทางเคลื่อนไหวตามเวลา ตัวเลือก ข้อมูลอินพุตอาจมีการบัฟเฟอร์โดยฟิลด์ฺซึ่งจะสร้างเป็นพื้นที่จากแต่ละตำแหน่ง จุดที่มีบัฟเฟอร์ หรือ อินพุตเป็นพื้นที่, จะแสดงผลต่อเนื่องตามลำดับสร้างเป็นพื้นที่โดยความกว้างแสดงค่าจากแอตทริบิวต์ที่สนใจ แทร็กที่ได้จะมีจุดเริ่มต้นและสิ้นสุดตามเวลาซึ่งเป็นตัวแทนของฟิเจอร์ชิ้นแรกและชิ้นสุดท้ายของแทร็ก เมื่อแทร็กที่ถูกสร้างขึ้น ค่าสถิติเกี่ยวกับฟีเจอร์ของอินพุตจะมีการคำนวณและส่งไปยังแทร็กเอาท์พุต สถิติพื้นฐานที่สุดคือการนับจำนวนของจุดในพื้นที่ แต่สถิติอื่น ๆ สามารถคำนวณได้ด้วย

ฟีเจอร์ในชั้นข้อมูลที่เปิดใช้เวลา สามารถแสดงในหนึ่งในสองวิธี ดังนี้:

  • สิ่งที่เกิดขึ้น - ครั้งเดียวในช่วงเวลา
  • สิ่งที่เกิดขึ้น – ตั้งแต่เริ่มจนสิ้นสุดเวลา

ตัวอย่างเช่น สมมติว่าคุณมีการวัดจีพีเอสของพายุเฮอริเคนทุก 10 นาที แต่ละเรคอร์ดที่วัดจีพีเอสบันทึกชื่อพายุเฮอริเคน, สถานที่, เวลาของการบันทึกและความเร็วลม ด้วยข้อมูลนี้คุณสามารถสร้างแทร็กสำหรับแต่ละพายุเฮอริเคน โดยใช้ชื่อในการระบุติดตาม และติดตามสำหรับพายุเฮอริเคนแต่ละเส้นจะถูกสร้างขึ้น นอกจากนี้คุณยังสามารถคำนวณสถิติเช่นค่าเฉลี่ยสูงสุดและความเร็วลมต่ำสุดของแต่ละพายุเฮอริเคนเช่นเดียวกับจำนวนจุดในแต่ละแทร็ค

โดยใช้ตัวอย่างเดียวกันคุณสามารถบัฟเฟอร์แทร็กของคุณโดยความเร็วลม บัฟเฟอร์แสดงความเร็วลมที่ตำแหน่งนั้น และต่อรวมกันเป็นพื้นที่บัฟเฟอร์ สร้างเป็น พื้นที่แทนเส้นทางการติดตามเช่นเดียวกับการเปลี่ยนแปลงในความเร็วลมเป็นพายุเฮอริเคนที่มีการเปลี่ยนแปลง

เลือกข้อมูลเพื่อสร้างจาก

ชั้นข้อมูลจุดหรือ พื้นที่ จะถูกสร้างขึ้นใหม่ลงในแทร็ก ชั้นข้อมูลอินพุตต้องเป็นเวลาที่เปิดใช้งานพร้อมกับคุณลักษณะที่แสดงถึงเวลาในทันที ชั้นข้อมูลต้องอยู่ในระบบพิกัดโปรเจคชัน หรือกำหนดผลการวิเคราะห์เป็นระบบพิกัดโปรเจคชัน โดยกำหนดด้วย สภาพแวดล้อมการวิเคราะห์ ถ้าการสร้างระยะกันชนถูกใช้งาน

นอกเหนือจากการเลือกชั้นข้อมูลจากแผนที่คุณสามารถเลือก เบราว์เลเยอร์ ที่ด้านล่างของรายการแบบเลื่อนลงเพื่อเรียกดูเนื้อหาของคุณสำหรับชุดข้อมูลชุดข้อมูลข้อมูลขนาดใหญ่หรือฟีเจอร์เลเยอร์

เลือกฟิลด์หนึ่งหรือมากกว่าเพื่อระบุแทร็ก

ฟิลด์ที่ใช้ระบุแทร็ก

ตัวอย่างเช่นถ้าคุณกำลังสร้างแทร็กของพายุเฮอริเคน คุณสามารถใช้ชื่อพายุเฮอริเคนเป็นฟิลด์แทร็ก

เลือกวิธีการที่ใช้ในการสร้างแทร็ก

วิธีการที่ใช้ในการรวมแทร็กและใช้บัฟเฟอร์ (ถ้ามี) วิธีระนาบ อาจคำนวณผลได้รวดเร็วยิ่งขึ้น แต่จะไม่ตัดแทร็กรอบเส้นแบ่งเขตวันสากลหรือลักษณะตามรูปร่างที่แท้จริงของโลกเมื่อบัฟเฟอร์ จีโอเดสิค วิธีการจะตัดแทร็กที่บริเวณเส้นวันหากมีความจำเป็นและใช้เนื้อที่บัฟเฟอร์ตามลักษณะของรูปร่างของแผ่นดิน

สร้างสมการสำหรับบัฟเฟอร์อินพุต โดยใช้ (ส่วนเสริม)

สมการที่ใช้ในการคำนวณระยะทางกันชนรอบฟีเจอร์ที่ใช้เป็นอินพุต สมการนี้สามารถสร้างขึ้นโดยใช้เครื่องคำนวณ บัฟเฟอร์และการดำเนินงานขั้นพื้นฐาน เช่น การ บวก ลบ คูณและหารด้วย ค่าที่คำนวณโดยใช้การวิเคราะห์ระบบพิกัด ชั้นข้อมูลต้องอยู่ในระบบพิกัดโปรเจคชัน หรือกำหนดผลการวิเคราะห์เป็นระบบพิกัดโปรเจคชัน โดยกำหนดด้วย สภาพแวดล้อมการวิเคราะห์ ถ้าการสร้างระยะกันชนถูกใช้งาน

สมการ เช่น $feature.windspeed * 1000 จะใช้บัฟเฟอร์ของ 1,000 ที่คูณด้วยฟิลด์ windspeed คุณอาจจะใช้มากกว่าหนึ่งฟิลด์ข้อมูลในเครื่องคิดเลขบัฟเฟอร์ได้

คุณยังสามารถระบุนิพจน์การรับรู้การติดตาม ตัวอย่างเช่น คุณสามารถคำนวณผลรวมของค่าแรก windspeed สำหรับฟีเจอร์ปัจจุบัน และสองฟีเจอร์เดิมด้วยสมการอย่างเช่น $track.field(windspeed).history(-3) ณ แต่ละตำแหน่ง ผลรวมของ windspeed ปัจจุบัน และการวัดสองครั้งก่อนหน้าจะถูกคำนวณและบัฟเฟอร์

เลือกเวลาที่จะแยกแทร็ก โดยใช้ (ส่วนเสริม)

เวลาที่ใช้ในการแยกแทร็ก หากจุดอินพุตหรือ พื้นที่มีระยะเวลานานกว่าระยะเวลาระหว่างที่แต่ละจุดจะแบ่งเวลา จุดเหล่านั้นจะถูกแบ่งออกเป็นแทร็กที่แตกต่างกัน

หากคุณระบุการแบ่งเวลาและการแบ่งระยะทาง แทร็กจะถูกแบ่งเมื่อตรงตามเงื่อนไขหนึ่งหรือสองเงื่อนไข

ลองจินตนาการว่าคุณมีฟีเจอร์จุดคิดเป็นเที่ยวบินเครื่องบินที่สนามติดตามเป็น ID เครื่องบิน เครื่องบินลำนี้จะทำให้การเดินทางหลายครั้งและจะแสดงเป็นหนึ่งแทร็ค ถ้าคุณรู้ว่ามีช่วง 1 ชั่วโมงระหว่างเที่ยวบินที่คุณสามารถใช้ 1 ชั่วโมง เป็นตัวแบ่งแต่ละเที่ยวบิน เที่ยวบันจะถูกแบ่งออกเป็นแทร็คของตัวเอง

เลือกระยะทาง เพื่อแยกแทร็กตาม (เลือกได้)

ระยะทางที่นำมาใช้เพื่อแยกแทร็ก หากจุดอินพุตหรือ พื้นที่มีระยะทางกว้างกว่าระยะทางระหว่างที่แต่ละจุดจะแบ่งเวลา จุดเหล่านั้นจะถูกแบ่งออกเป็นแทร็กที่แตกต่างกัน

หากคุณระบุการแบ่งเวลาและการแบ่งระยะทาง แทร็กจะถูกแบ่งเมื่อตรงตามเงื่อนไขหนึ่งหรือสองเงื่อนไข

เพิ่มสถิติ (อุปกรณ์เสริม)

คุณสามารถคำนวณสถิติเกี่ยวกับคุณสมบัติที่มีรายละเอียด คุณสามารถคำนวณรายการต่อไปนี้บนฟิลด์ตัวเลข:

  • นับ - นับจำนวนข้อมูลที่ไม่ใช่ค่าว่าง ซึ่งนำมาใช้ได้ทั้งในฟิลด์ตัวเลข หรือตัวอักษร นับจำนวนของ [null, 0, 2] คือ 2
  • รวม - ผลรวมของค่าตัวเลขในฟิลด์ ผลรวมของ [null, null, 3] คือ 3
  • เฉลี่ย - ค่าเฉลี่ยของค่าตัวเลข ค่าเฉลี่ยของ [0, 2, null] เท่ากับ 1
  • ต่ำสุด - ค่าต่ำสุดของฟิลด์ตัวเลข ค่าต่ำสุดของ [0, 2, null] เท่ากับ 0
  • สูงสุด - ค่าสูงสุดของฟิลด์ตัวเลข ค่าสูงสุดของ [0, 2, null] คือ 2
  • ช่วง - ช่วงของฟิลด์ตัวเลข ค่านี้คำนวณจากค่าสูงสุด ลบด้วยค่าต่ำสุด ช่วงของ [0, null, 1] คือ 1 ช่วงของ [null, 4] คือ 0
  • ความแปรปรวน - ความแปรปรวนของฟิลด์ตัวเลขในแทร็ก ความแปรปรวนของ [1] เป็น null ความแปรปรวนของ [null, 1,0,1,1] เท่ากับ 0.25
  • ค่าเบี่ยงเบนมาตรฐาน - ค่าเบี่ยงเบนมาตรฐานของฟิลด์ตัวเลข ค่าเบี่ยงเบนมาตรฐานของ [1] เป็น null ค่าเบี่ยงเบนมาตรฐานของ [null, 1,0,1,1] เท่ากับ 0.5

คุณสามารถคำนวณรายการต่อไปนี้บนฟิลด์ตัวอักษร:

  • นับ – นับจำนวนข้อมูลสตริงที่ไม่ใช่ค่าว่าง
  • ค่าใด ๆ ต่อไปนี้ เป็นสถิติตัวอย่างที่สุ่มจากค่าสตริงในฟิลด์ที่ระบุ
สถิติทั้งหมดจะมีการคำนวณค่า nonnull ชั้นข้อมูลที่เกิดขึ้นจะมีข้อมูลใหม่สำหรับแต่ละสถิติคำนวณ จำนวนสถิติใด ๆ ที่สามารถเพิ่มได้โดยการเลือกคุณลักษณะและสถิติ

เลือกที่เก็บข้อมูล ArcGIS ดาต้าสโตร์ เพื่อบันทึกผลลัพธ์

GeoAnalytics ผลลัพธ์จะถูกเก็บไว้ใน ArcGIS Data Store และถูกสร้างเป็นชั้นข้อมูลฟีเจอร์ ใน Portal for ArcGIS ในกรณีส่วนใหญ่ ผลลัพธ์ควรเก็บไว้ในที่เก็บข้อมูล spatiotemporal และใช้เป็นค่าเริ่มต้นด้วย ในบางกรณี การบันทึกผลลัพธ์ ไปยังที่เก็บข้อมูลเชิงสัมพันธ์เป็นตัวเลือกที่ดี ต่อไปนี้คือเหตุผลที่คุณอาจต้องการจัดเก็บผลลัพธ์ ไว้ในการเก็บข้อมูลเชิงสัมพันธ์:

  • คุณสามารถ ใช้ผลลัพธ์ในพอร์ทัล-เพื่อ-การทำงานร่วมกันของพอร์ทัล
  • คุณสามารถเปิดใช้งานความสามารถในการซิงค์กับผลลัพธ์ของคุณ

คุณไม่ควรใช้ที่เก็บข้อมูลเชิงสัมพันธ์ ถ้าคุณคาดว่าจะนำผลลัพธ์จาก GeoAnalytics ของคุณไปใช้ประโยชน์เพิ่มขึ้น และจำเป็นต้องใช้ประโยลน์จากความสามารถในการจัดเก็บข้อมูลขนาดใหญ่ของ spatiotemporal เพื่อจัดการกับข้อมูลจำนวนมาก

ชื่อของชั้นข้อมูลผลลัพธ์

ชื่อ ของชั้นข้อมูลที่จะถูกสร้างขึ้นใน เนื้อหาของฉัน และเพิ่มไปยังแผนที่ ชื่อเริ่มต้นจะขึ้นอยู่กับชื่อเครื่องมือและชื่อชั้นข้อมูลอินพุต ถ้าชั้นข้อมูลมีอยู่แล้วเครื่องมือจะล้มเหลว

ใช้ บันทึกผลลัพธ์ใน รายการตัวเลือก เพื่อระบุชื่อของโฟล์เดอร์ใน เนื้อหาของฉัน เมื่อบันทึกผล
